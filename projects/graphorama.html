Graphorama

Graphorama is a WPF-based function plotting application written in C#. It uses the OxyPlot library to graph mathematical expressions in real-time, with support for multiple simultaneous plots, axis adjustments, and basic input validation. It does exactly what you think it does... it draws lines based on equations. Not much more, not much less.

Functionality Overview

  • Plots mathematical functions using OxyPlot
  • Supports basic arithmetic, trig, logs, and exponentials
  • Multiple expressions can be plotted at once
  • Adjustable axis limits with live updates
  • Real-time graphing toggle with user-defined speed control
  • Input validation for malformed expressions

Technical Architecture

The application is built in C# using WPF for the user interface and OxyPlot for rendering. Expression parsing is handled internally, with basic safeguards to avoid runtime crashes when someone inevitably types "log()" with no arguments. The structure is straightforward: a text box, a graph, and some buttons. It does what it needs to.

Technologies: C#, WPF, OxyPlot, .NET 8.0

Real-Time Plotting

When real-time mode is enabled, functions are plotted progressively over time. This feature exists mostly to make the plotting experience feel more dynamic. Speed can be adjusted within a reasonable range. Values outside that range may break things. That's on you.

Input Format

Function inputs accept typical math expressions. Example syntax:

  • 2x + 3 - Linear
  • sin(x) - Trig
  • log(x) - Logarithmic
  • ArcTan(x) - Inverse trig (case-sensitive)
  • sqrt(x) - Square root

Syntax errors will prompt a warning message. Some functions are case-sensitive by design, which is either a technical limitation or a test of patience.

Graph Management

  • Update axis limits manually using X Min and X Max fields
  • Clear the graph with one button click
  • Select plotted functions from the list to delete
  • Zoom and pan with mouse input (assuming OxyPlot doesn’t crash)
